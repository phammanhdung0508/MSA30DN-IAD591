## 2026-03-11 - SQL Injection in Dynamic Intervals and Insecure CORS
**Vulnerability:** SQL injection was possible via f-string interpolation in SQLite `datetime` and `strftime` functions used for dynamic time intervals. Additionally, the FastAPI backend was configured with `allow_credentials=True` and `allow_origins=["*"]`, which is insecure and disallowed by modern browsers.
**Learning:** Developers often forget that dynamic parts of a SQL query like intervals ('-24 hours') still need to be parameterized even if they are not direct column values. Parameterizing the value and using SQLite's string concatenation (`||`) within the query is a safe way to handle this. For CORS, the combination of wildcard origins and credentials allowed is a common misconfiguration that browsers block for security reasons.
**Prevention:** Always use `?` placeholders for all dynamic data in database queries, including values passed to SQL functions. Maintain a strict CORS policy and never enable `allow_credentials` when using `allow_origins=["*"]`.
